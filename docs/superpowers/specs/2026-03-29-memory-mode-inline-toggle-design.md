# MemoryMode 底部按钮原地切换设计

## 目标
修正 `MemoryMode.vue` 中底部操作栏的“收藏”和“掌握”按钮行为，使点击按钮时只切换当前单词状态，不再自动切换到下一张卡片。

## 已确认的交互
- 底部“收藏”按钮：切换当前单词的收藏状态，支持取消收藏。
- 底部“掌握”按钮：切换当前单词的掌握状态，支持取消掌握。
- 底部按钮点击后不触发飞出动画，不修改 `currentIndex`，也不强制翻转卡片。
- 右滑手势仍表示收藏当前单词并切换到下一张。
- 左滑手势仍表示掌握当前单词并切换到下一张。

## 设计决策
- 保留现有 `handleFavorite()` / `handleMastered()` 作为手势路径，继续负责动画和翻页。
- 新增仅供底部按钮点击的原地切换方法，避免在同一个方法里混入“是否翻页”的条件分支。
- `sessionStats` 在按钮取消状态时同步递减，避免统计只增不减。
